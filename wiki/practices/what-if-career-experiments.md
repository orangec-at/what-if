# What-If Career Experiments

## Summary

What-if career experiments are weekly, playful, artifact-producing quests for becoming a [[concepts/remote-first-frontend-fullstack-engineer|remote-first frontend-based fullstack engineer]].

Each experiment asks a question that feels interesting enough to pursue, then converts the answer into proof: code, design notes, demos, async documents, interview stories, or portfolio material.

## Procedure

1. **Draw the card** — choose one weekly what-if from the quest map or write a new one with `templates/what-if-card.md`.
2. **Build tiny proof** — create the smallest artifact that answers the question.
3. **Add remote signal** — include an async note, tradeoff log, PR summary, demo script, or decision record.
4. **Polish one visible thing** — improve only the most visible surface.
5. **Ship the loot** — turn the result into a portfolio section, GitHub link, resume bullet, interview answer, or private wiki page.
6. **Log XP** — record what improved and what the next what-if should be.

## First quest

> What if I could prove remote trust before the first interview?

Suggested artifact package:

- Async feature proposal
- Tradeoff note
- PR-style summary
- Demo script
- Portfolio-ready “How I work async” section

## Guardrails

- One week, one card.
- Artifact beats study notes.
- Fun is not decoration; it is the engine.
- Avoid huge plans. Shrink until the quest feels playable.
- Private artifacts count. Public posting is optional.
- Rejection is telemetry, not identity damage.
