# Remote-First Frontend-Based Fullstack Engineer

## Summary

A remote-first frontend-based fullstack engineer is a [[concepts/software-developer|software developer]] who leads from user experience and product feel, then connects the API, data model, authentication, deployment, tests, and documentation needed to ship the product slice.

The target identity is not “frontend plus a little backend.” It is:

> Someone who can make a product idea visible, usable, shippable, and easy to collaborate on asynchronously.

## Why it matters

Remote-first companies hire for more than coding ability. They need people who can reduce ambiguity, communicate decisions clearly, and create trustworthy progress without constant synchronous supervision.

This identity gives the career system a concrete north star:

> I am recognized as a frontend-based fullstack engineer who can own product slices end-to-end in a remote-first environment.

## Capability axes

| Axis | Hiring signal | Example artifact |
| --- | --- | --- |
| Frontend Taste | UI polish, interaction quality, accessibility, performance, product thinking | polished demo, component breakdown, before/after UI note |
| Fullstack Ownership | API, database, auth, error states, tests, deployment | end-to-end feature, architecture note, API route docs |
| Remote Trust | async communication, decision logs, tradeoffs, progress visibility | async proposal, PR-style summary, demo script, decision record |
