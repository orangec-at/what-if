# Remote Guild Quest System

## Summary

The Remote Guild Quest System reframes career preparation as a playful sequence of weekly what-if quests. The user is not merely job hunting; they are preparing to enter a remote-first guild as a [[concepts/remote-first-frontend-fullstack-engineer|frontend-based fullstack adventurer]].

Each quest starts with a fun what-if question and ends with tangible career loot: a portfolio artifact, GitHub proof, interview story, resume bullet, or remote collaboration signal.

## Why it matters

The system protects motivation. It avoids turning the job search into a heavy productivity ritual while still producing serious evidence that a remote-first hiring team can understand.

The rule is simple:

> If it does not produce an artifact or make the next quest more fun, shrink it.

## Game vocabulary

| Term | Meaning | Career translation |
| --- | --- | --- |
| Quest | One small weekly what-if experiment | A bounded artifact-producing task |
| XP | Skill or confidence gained | Frontend, fullstack, remote trust, storytelling |
| Loot | Reusable output | Resume bullet, portfolio section, GitHub link, interview story |
| Boss fight | High-pressure checkpoint | Interview, take-home, recruiter call, offer negotiation |
| Telemetry | Signal from reality | Rejection, feedback, recruiter response, demo reaction |

## Core loop

1. Draw a what-if card.
2. Build tiny proof.
3. Add remote signal.
4. Polish one visible thing.
5. Ship the loot.
6. Log XP.
7. Choose the next what-if.
