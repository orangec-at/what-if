# Entity Types

| Type | Use for | Typical directory |
| --- | --- | --- |
| `concept` | Ideas, mental models, domains, principles when not normative enough for `principle`. | `wiki/concepts/` |
| `practice` | Repeatable developer activities or workflows. | `wiki/practices/` |
| `tool` | Languages, frameworks, libraries, CLIs, services, editors, runtimes. | `wiki/tools/` |
| `artifact` | Things produced or maintained: tests, ADRs, indexes, logs, APIs. | `wiki/artifacts/` |
| `principle` | Normative guidance or heuristics. | `wiki/concepts/` or future `wiki/principles/` |
| `quality-attribute` | Maintainability, reliability, performance, security, usability. | future `wiki/qualities/` |
| `risk` | Failure modes, hazards, coordination problems, security issues. | future `wiki/risks/` |
| `decision` | Chosen approach and rationale. | `wiki/decisions/` |
| `source` | Summary of raw source or URL. | `wiki/sources/` |
| `synthesis` | Cross-page answer, map, comparison, or thesis. | `wiki/syntheses/` |
| `question` | Durable user question and answer. | `wiki/questions/` |
| `lint-report` | Wiki health-check results. | `wiki/lint/` |
