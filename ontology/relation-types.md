# Relation Types

| Relation | Meaning | Example |
| --- | --- | --- |
| `is-a` | Taxonomic membership. | `React is-a frontend framework`. |
| `part-of` | Component or subdomain relationship. | `Unit testing part-of testing`. |
| `depends-on` | Requires or assumes another entity. | `CI depends-on test suite`. |
| `enables` | Makes something easier or possible. | `Index enables query`. |
| `constrains` | Limits or shapes another entity. | `Latency budget constrains architecture`. |
| `implements` | Concrete realization of an idea or decision. | `Folder layout implements LLM-maintained wiki`. |
| `tested-by` | Verified by a test or check. | `Wiki links tested-by wiki_lint.py`. |
| `mitigates` | Reduces a risk. | `Code review mitigates defect risk`. |
| `trades-off-with` | Improves one thing while worsening another. | `Caching trades-off-with freshness`. |
| `supersedes` | Replaces an older claim or decision. | `New source supersedes old benchmark`. |
| `contradicts` | Conflicts with another claim/source/page. | `Source A contradicts Source B`. |
| `evidence-for` | Supports a claim. | `Source summary evidence-for LLM Wiki pattern`. |
| `observed-in` | Claim appears in a source or context. | `Practice observed-in source`. |
| `asked-by` | Question originated from a user/session. | `Question asked-by user`. |
| `answered-by` | Question answered by a synthesis/source. | `Question answered-by synthesis`. |
